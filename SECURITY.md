# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Authrim, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

### How to Report

Send an email to: **yuta@sgrastar.org**

Please include:
- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested fixes (optional)

### What to Expect

- **Acknowledgment**: Within 48 hours of your report
- **Initial Assessment**: Within 7 days
- **Resolution Timeline**: Depends on severity and complexity

### Scope

This security policy applies to:
- The Authrim core codebase
- Official packages in this repository

Out of scope:
- Third-party dependencies (report to their maintainers)
- Issues in user deployments or configurations

### Disclosure Policy

- We follow responsible disclosure practices
- We will coordinate with you on disclosure timing
- Credit will be given to reporters (unless anonymity is requested)

## Security Status

> **Note**: Authrim is currently in development and **not production-ready**.
> Security hardening and formal audits have not been completed.
> See [README.md](README.md) for current project status.
